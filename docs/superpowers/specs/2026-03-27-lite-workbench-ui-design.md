# 轻工作台 UI 收敛设计

## 背景
- 当前前端存在明显“演示页 / 讲解页”倾向，页面文案和信息架构过度强调范式说明、演示路径和面试叙事。
- 用户希望把界面收回到更朴素的工作台形态：重点展示用户输入、模型输出、步骤结果，以及清晰可达的配置入口。

## 用户确认结论
- 保留 `Overview`，但改成极简工作台首页。
- `Overview` 首页需要显示最近一次运行结果。
- `Playground` 改成纯运行台，只保留 `Agent + 输入 + 输出 + 步骤`。
- `Plan and Execute / ReAct / Reflection` 不再放在主界面里。
- 页面文案改成中性、朴素、可用，不再强调“演示 / 面试 / 讲解”。

## 目标
- 让首页成为真实可用的入口页，而不是介绍页。
- 让 `Playground` 成为最短运行路径。
- 让 `Settings` 成为清晰的配置汇总入口。
- 降低整体视觉和文案的浮夸感，保留已有能力但减少“叙事包袱”。

## 设计决策

### 1. Overview 改为轻工作台
- 顶部保留一行简短说明和 2~3 个快捷操作按钮。
- 主体改为均衡工作台布局：最近一次运行、系统状态、常用入口、配置快捷入口。

### 2. Playground 改为纯运行台
- 删除范式选择卡片、讲解型提示和“演示提示”输入框。
- 仅保留 Agent 选择、最大轮次、输入框、运行按钮、最终输出、步骤列表。
- 保留当前失败态和空结果态修复，不再出现整块空白。

### 3. Settings 保持清晰配置入口
- 保留现有分组卡片结构。
- 将所有“主叙事 / 演示入口 / 支撑能力”式文案改为纯配置语言。

### 4. 导航与全局壳层收敛
- 一级导航只保留 `工作台 / 运行台 / 知识库 / 运行记录 / 配置`。
- `Workflows`、`Tools` 降级为非一级入口，保留路由兼容但不作为主导航重点。
- 页头副标题和按钮改成中性表述，例如“开始运行”“打开配置”。

### 5. 视觉风格收敛
- 减弱大面积渐变、过强阴影和过多 Hero 文案。
- 统一成轻量卡片 + 清晰留白 + 中性色背景。
- 保持现有技术栈和组件，不做大规模视觉系统迁移。

## 范围边界
- 不改后端接口契约。
- 不新增长链路观测功能。
- 不重做所有管理页，仅调整壳层、首页、运行台和文案。

## 验收标准
- 首页第一屏不再出现“演示 / 面试 / 讲解”导向文字。
- `Playground` 首屏只出现运行相关控件和结果区域。
- 顶部导航更接近日常使用工作台，不再突出 `Workflows / Tools`。
- 前端构建通过，原有主要路由仍可访问。
