# 测试提示词

这份只保留**最有信号**的测试项，用来快速验证：固定对话、Auto、Flow、RAG、MCP、异常处理。

## 1. Fixed：基础链路

- **测试功能**：固定配置 / 普通对话
- **提示词**：`请用三句话介绍 Spring AI。`
- **重点观察**：输出是否直接、稳定，不要出现工具调用和多阶段编排痕迹。

## 2. Auto：监督可能不通过

- **测试功能**：分析 / 执行 / 监督 / 多轮反馈
- **提示词**：`请给我 10 个 AI Agent 项目亮点，每个都必须包含：技术实现、业务价值、边界限制、面试官追问点、可量化成果，少一项都不合格。`
- **重点观察**：是否出现监督不通过、重新分析、再次执行。

## 3. Flow：结构化规划

- **测试功能**：工具分析 / JSON 规划 / 分步执行
- **提示词**：`先查深圳天气，再搜深圳大学粤海校区附近咖啡店，最后给我一个简短出行建议。`
- **重点观察**：是否先出现 planning，再按步骤执行；前端 planning 阶段是否能看到结构化步骤。

## 4. RAG：标签检索

- **测试功能**：知识库检索增强
- **前置条件**：先上传一份带明确知识标签的文档。
- **提示词**：`根据“Java面试”知识库，简要总结一下 Spring Boot 自动配置机制。`
- **重点观察**：是否命中正确知识标签，回答是否明显带知识库内容，而不是泛泛而谈。

## 5. MCP：工具是否真的调用

- **STDIO 提示词**：`必须使用 filesystem MCP 工具读取 docs/mcp_probe.txt，只返回文件名和测试码。`
- **Amap 提示词**：`必须使用 amap_weather 工具查询上海今天天气，只返回核心结果。`
- **重点观察**：后端日志是否出现 MCP tool invocation；回答是否克制、不猜测。

## 6. 异常处理

- **测试功能**：前后端错误提示
- **测试方式**：Agent ID 填一个不存在的值，比如 `999`
- **重点观察**：前端是否提示错误、不会一直 loading；后端是否有清晰异常日志。

## 推荐顺序

1. 先测 **Fixed**，确认基础链路正常。
2. 再测 **Auto** 和 **Flow**，确认编排链路正常。
3. 再测 **RAG** 和 **MCP**，确认增强能力正常。
4. 最后测 **异常处理**。
