# 测试提示词

用于联调时快速判断：**路由对不对、规划对不对、MCP 有没有真的调用、异常处理是否正常**。

## 1. ID1：固定回复测试

- **测试功能**：固定提示词 / 固定角色输出
- **提示词**：请用三句话介绍一下你自己。
- **重点看什么**：
  - 是否走固定配置
  - 输出是否稳定
  - 是否没有多余发挥

## 2. ID2：通用能力测试

### 2.1 学习规划

- **测试功能**：自动编排 / 结构化输出
- **提示词**：帮我做一个 Java 后端学习路线，分 4 周，每周给出重点和练习建议。
- **重点看什么**：
  - 是否能按周拆分
  - 是否有明确重点
  - 是否有练习建议

### 2.2 简单问答

- **测试功能**：普通问答 / 非工具调用
- **提示词**：请总结 Spring AI 和传统 SDK 调用的区别。
- **重点看什么**：
  - 是否直接回答
  - 是否条理清楚
  - 是否没有乱扩展

### 2.3 多步骤计划

- **测试功能**：多步骤规划能力
- **提示词**：给我一个 7 天的 RAG 学习计划，每天 1 小时。
- **重点看什么**：
  - 是否按天拆分
  - 是否有阶段性安排
  - 是否符合“每天 1 小时”约束

### 2.4 约束型任务

- **测试功能**：约束理解能力
- **提示词**：帮我做一个适合转岗 Java 后端的学习计划，预算 0 元，周期 1 个月。
- **重点看什么**：
  - 是否满足“0 元”
  - 是否满足“1 个月”
  - 是否真的面向转岗场景

### 2.5 异常处理

- **测试功能**：前端错误提示 / 后端异常处理
- **测试方式**：Agent ID 填不存在的值，例如 `999`
- **重点看什么**：
  - 前端是否提示错误
  - 是否不会一直转圈
  - 后端是否有清晰报错

## 3. ID3：Flow / MCP 测试

- **测试功能**：Flow 编排 + MCP 工具调用
- **SSE 启动命令**：`python tools/amap_sse_mcp.py`

### 3.1 STDIO：文件系统 MCP

- **测试功能**：`filesystem` 工具调用
- **提示词**：必须使用 `filesystem` MCP 工具读取 `docs/mcp_probe.txt`，只返回文件名和测试码，不要猜测，不要补充其他内容。
- **重点看什么**：
  - 是否真的调用了 `filesystem`
  - 是否只返回文件名和测试码
  - 是否没有模型自由发挥

### 3.2 SSE：天气查询

- **测试功能**：`amap_weather` 工具调用
- **提示词**：必须使用 `amap_weather` 工具查询上海今天天气，只返回核心结果，不要猜测。
- **重点看什么**：
  - 是否真的调用了 `amap_weather`
  - 是否返回天气核心信息
  - 是否没有猜测性内容

### 3.3 SSE：POI 搜索

- **测试功能**：`amap_search_poi` 工具调用
- **提示词**：必须使用 `amap_search_poi` 工具查找杭州西湖附近咖啡店，只返回前 3 个结果。
- **重点看什么**：
  - 是否真的调用了 `amap_search_poi`
  - 是否只返回前 3 个结果
  - 是否结果简洁可读

## 4. 推荐测试顺序

1. 先测 `ID1`，确认基础链路正常。
2. 再测 `ID2`，确认普通问答、规划、约束处理正常。
3. 最后测 `ID3`，确认 MCP 工具链路正常。
