# 测试提示词

用于前后端联调、Agent 路由验证、MCP 工具调用验证。

## ID1：固定提示词

- **类型**：`fixed`
- **测试提示词**：请用三句话介绍一下你自己

---

## ID2：自动编排 / 通用能力测试

- **类型**：`auto`
- **测试提示词**：帮我做一个 Java 后端学习路线，分 4 周，每周给出重点和练习建议

### 简单问答

- **测试提示词**：请总结 Spring AI 和传统 SDK 调用的区别

### 多步骤计划

- **测试提示词**：给我一个 7 天的 RAG 学习计划，每天 1 小时

### 约束型任务

- **测试提示词**：帮我做一个适合转岗 Java 后端的学习计划，预算 0 元，周期 1 个月

### 异常测试

- **测试方式**：Agent ID 填不存在的值，例如 `999`
- **预期结果**：前端提示错误，而不是一直转圈

---

## ID3：Flow / MCP 工具测试

- **类型**：`flow`
- **SSE 启动命令**：`python tools/amap_sse_mcp.py`
- **测试范围**：SSE 与 STDIO

### STDIO 测试

#### 文件系统 MCP

- **测试提示词**：必须使用 `filesystem` MCP 工具读取 `docs/mcp_probe.txt`，只返回文件名和测试码，不要猜测，不要补充其他内容。
- **验证点**：
  - 是否实际调用了 `filesystem` MCP
  - 是否只返回文件名和测试码
  - 是否没有额外发挥

### SSE 测试

#### 天气查询

- **测试提示词**：必须使用 `amap_weather` 工具查询上海今天天气，只返回核心结果，不要猜测。
- **验证点**：
  - 是否实际调用了 `amap_weather`
  - 是否返回天气核心信息
  - 是否没有模型自行猜测

#### POI 搜索

- **测试提示词**：必须使用 `amap_search_poi` 工具查找杭州西湖附近咖啡店，只返回前 3 个结果。
- **验证点**：
  - 是否实际调用了 `amap_search_poi`
  - 是否限制为前 3 个结果
  - 是否结果格式简洁

---

## 建议用法

- 优先按 `ID1 -> ID2 -> ID3` 的顺序测试
- 做 MCP 测试前，先确认对应服务已经启动
- 做异常测试时，重点观察前端提示和接口返回是否一致
